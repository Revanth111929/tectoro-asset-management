🎯 STEP 1: CHECK REQUIREMENTS

   Open terminal and run:

   $ cd /home/administrator/asset-management
   $ ./check_requirements.sh

   This checks if Node.js and Python are installed.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🎯 STEP 2: AUTOMATED SETUP

   Run the setup script:

   $ ./START_HERE.sh

   This will:
   • Install all Python packages
   • Install all Node.js packages
   • Configure your IP address
   • Show you what to do next

   ⏱️  Takes 3-5 minutes

🎯 STEP 3: START BACKEND

   Open Terminal 1:

   $ cd /home/administrator/asset-management
   $ source venv/bin/activate
   $ python3 api_server.py

   You should see:
   ┌────────────────────────────────────────────────────────┐
   │ 🚀 Asset Management API Server                         │
   │ ✅ API running on: http://0.0.0.0:5000                 │
   └────────────────────────────────────────────────────────┘

🎯 STEP 4: START FRONTEND

   Open Terminal 2 (NEW terminal window):

   $ cd /home/administrator/asset-management/frontend
   $ npm start

   Browser will automatically open to:
   http://localhost:3000

   ✨ KEEP THIS TERMINAL OPEN TOO!

🐛 COMMON ISSUES:

   ❌ "npm: command not found"
      → Install Node.js (see SETUP_GUIDE.md)

   ❌ "Port 3000 already in use"
      → sudo kill -9 $(sudo lsof -t -i:3000)

   ❌ "Cannot connect to backend"
      → Check if Terminal 1 (backend) is running

   ❌ "Can't access from phone"
      → Check firewall: sudo ufw allow 3000/tcp && sudo ufw allow 5000/tcp

   For more solutions, see TROUBLESHOOTING.md

💡 PRO TIPS:

   • Keep both terminals open while working
   • Backend must start BEFORE frontend
   • Use your actual IP, not 127.0.0.1 or localhost
   • Test on same computer first, then try other devices
   • Press F12 in browser to see console (for debugging)
   • React auto-reloads when you save files
   • Flask needs manual restart after code changes
